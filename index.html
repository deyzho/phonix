Quick start

From zero to deployed in minutes

One CLI, one config file, one client API — regardless of which network you deploy to.

1

Initialise your project

Choose a provider and template. AxonSDK generates your config and entry point.

2

Configure credentials

The axon auth wizard generates and stores all required keys. Your .env is locked to owner-only permissions and never committed.

3

Test locally

Run axon run-local to simulate the full provider runtime on your machine — no network required.

4

Deploy and call

One command bundles and deploys your workload. Then call from any JavaScript environment with @axonsdk/sdk.

// Deploy once — run on GPU edge nodes
import { AxonClient } from '@axonsdk/sdk';

const client = new AxonClient({
  provider: 'ionet',
  secretKey: process.env.AXON_SECRET_KEY,
});

await client.connect();

// Listen for results from the GPU worker
client.onMessage((msg) => {
  const { result } = msg.payload as { result: string };
  console.log('Result:', result);
});

// Send a prompt to a processor node
await client.send('gpu-worker-0x1a2b3c', {
  requestId: 'req-001',
  prompt: 'Summarize: The quick brown fox...',
});

client.disconnect();

import {
  useAxonContext,
  useMessages,
  useSend,
} from '@axonsdk/mobile';

export function ResultsScreen() {
  const { connected } = useAxonContext();
  const messages = useMessages(client);
  const { send, sending } = useSend(client);

  return (
    <View>
      <Text>{connected ? '🟢 Live' : '⚪ Offline'}</Text>
      <Button
        title={sending ? 'Sending...' : 'Run inference'}
        onPress={() => send(leaseUrl, {
          prompt: 'Summarize the news today'
        })}
      />
      {messages.map((m, i) => (
        <Text key={i}>{m.payload as string}</Text>
      ))}
    </View>
  );
}
// Without context — manage client directly
import { useAxon, useMessages } from '@axonsdk/mobile';

export function InferenceScreen() {
  const { client, connected, connect, error }
    = useAxon({
        provider: 'akash',
        secretKey: AXON_SECRET_KEY,
      });
  const messages = useMessages(client);

  return (
    <View>
      <Button
        title="Connect"
        onPress={connect}
        disabled={connected}
      />
      {error && <Text>{error.message}</Text>}
      {messages.map((m, i) => (
        <Text key={i}>
          {JSON.stringify(m.payload)}
        </Text>
      ))}
    </View>
  );
}

Smart Routing

Route across providers — automatically

AxonRouter manages multiple providers simultaneously. On every request it scores each provider by health, latency, cost, and availability — then sends to the best one. If that provider fails, it retries on the next. Circuit breakers prevent routing to unhealthy nodes until they recover.

  • balancedEqual weight across availability, latency, and cost — the safe default.
  • latencyAlways picks the fastest-responding provider. Good for interactive workloads.
  • availabilityMaximises uptime. Prefers providers with the highest recent success rate.
  • costRoutes to the cheapest option. Ideal for batch and background jobs.
  • round-robinDistributes load evenly across all callable providers.

Works on Node.js (SDK) and React Native (mobile). The same routing engine powers both. 

// Server SDK
import { AxonRouter } from '@axonsdk/sdk';

const router = new AxonRouter({
  providers: ['akash', 'acurast'],
  secretKey: process.env.AXON_SECRET_KEY,
  strategy: 'latency',
});

await router.connect();
await router.deploy(config);  // deploys to all providers

// Automatically picks fastest callable provider
await router.send({ prompt: 'Hello' });

// Health snapshot
router.health().forEach(h => {
  console.log(h.provider, h.latencyMs, h.circuitState);
});
// Mobile (React Native / Expo)
import { useAxonRouter } from '@axonsdk/mobile';

const { router, connected, health } = useAxonRouter({
  routes: [
    { provider: 'akash',   endpoint: AKASH_URL,   secretKey },
    { provider: 'acurast', endpoint: ACURAST_WS, secretKey },
  ],
  strategy: 'balanced',
  autoConnect: true,
});

// AppState-aware: pauses on background, resumes on foreground
await router?.send({ prompt: 'Hello from iOS' });
Features

Built for production

Security and reliability are not afterthoughts — built into every layer of the SDK and covered by 200+ tests across all ten providers.

🔒

Confidential by default

Your code runs inside hardware TEEs. Prompts, responses, and logic are private — even from device owners and network operators.

🛡️

SSRF & DNS rebinding protection

All HTTP calls validate URLs and resolve hostnames to IPs before opening connections, blocking requests to internal infrastructure.

🔑

Safe credential management

The auth wizard generates keys locally, writes them with chmod 600, and enforces .gitignore — secrets never leave your machine accidentally.

📦

Single-file bundles

esbuild compiles your TypeScript to a single optimised IIFE with env vars injected at build time. No runtime dependencies on the edge node.

🧪

Local testing runtime

The mock runtime simulates the full provider API locally — WebSocket messages, HTTP callbacks, fulfill — so you iterate without touching the network.

🌐

Provider-agnostic API

Switch from io.net to Akash to Acurast by changing one config field. Your application code stays identical across all five providers.

🖥️

GPU compute via io.net

A100 and H100 spot clusters at ~$0.40/hr. The same SDK interface — deploy, send, receive — with a 60-second timeout and 4 MiB response cap for large model outputs.

🔌

OpenAI-compatible endpoint

Drop @axonsdk/inference into any Next.js, Express, or Cloudflare Worker. Change baseURL in your existing OpenAI client and you're done.

📱

iOS & Android ready

@axonsdk/mobile brings React hooks, context, and AppState lifecycle to your React Native and Expo apps — iOS Keychain, Android Keystore, zero native modules.

⚡

Intelligent multi-provider routing

AxonRouter scores every provider on latency, availability, and cost in real time. Circuit breakers open on failure and recover automatically — zero downtime across all providers.
